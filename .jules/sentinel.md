## 2024-05-18 - [Insecure Direct Object Reference (IDOR) & Missing Authentication on Post Endpoints]
**Vulnerability:** The application was vulnerable to an IDOR where a user could specify `userId` in the request body to create posts on behalf of another user. Furthermore, the `POST /api/posts/` and `POST /api/posts/:id/react` endpoints lacked the `authenticateToken` middleware, allowing completely unauthenticated access.
**Learning:** Endpoints that mutate data or are expected to be performed by an authenticated user must explicitly have authentication middleware applied. Additionally, user identity should never be trusted from the client payload (like `req.body.userId`) when performing sensitive actions; it must be extracted from the authenticated session (e.g., `req.user.id`).
**Prevention:** Always verify that routes have the necessary authorization/authentication middleware. For resource creation and modification, rely exclusively on trusted context objects (such as tokens/sessions stored in `req.user`) instead of client-provided IDs.
