## 2024-05-24 - [CRITICAL] Prevented IDOR in Post Creation
**Vulnerability:** IDOR vulnerability in the `POST /api/posts` endpoint where the backend accepted an unverified `userId` from the request body to create a post, allowing users to impersonate others by spoofing the `userId`. The post routes were also entirely unprotected without authentication middleware. In addition, the `auth.middleware.js` incorrectly referenced `decoded.userId` when generating req.user object instead of `decoded.id` that is used to sign the token.
**Learning:** Endpoints creating user-associated resources must enforce authentication, and the resource owner should be inferred reliably from server-side session data (like the authenticated token), not user-provided input. Also always ensure that the signed JWT claims match the expected parsed claims inside the authorization middleware.
**Prevention:** Rely on `req.user.id` assigned by authentication middleware instead of `req.body.userId`. Make sure the auth middleware correctly extracts the user `id` field. Secure endpoints with middleware to ensure operations are authorized and valid.
