## 2024-05-17 - [IDOR in Post Creation Endpoint]
**Vulnerability:** Any user could create a post on behalf of any other user because the `createPost` API endpoint blindly accepted a `userId` from the request body rather than authenticating it. There was also no authentication requirement on the `POST /api/posts/` and `POST /api/posts/:id/react` routes.
**Learning:** This codebase lacked middleware protecting critical user endpoints by default. Furthermore, test environments shared the same external PostgreSQL instance rather than utilizing an isolated SQLite database, which could lead to test data interfering with dev/prod data.
**Prevention:** In this codebase, always ensure that `req.user.id` (set by the `authenticateToken` middleware) is used instead of client-provided IDs (like `req.body.userId`) for resource creation. Verify that `authenticateToken` is applied on all non-public modification endpoints. Configure `NODE_ENV=test` to use an in-memory SQLite database (`sqlite::memory:`) in `backend/src/config/db.js` to ensure safe, isolated test environments.
