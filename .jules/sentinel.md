## 2024-05-18 - [CRITICAL] Fix unauthenticated post creation and IDOR
**Vulnerability:** The POST `/api/posts/` and `/api/posts/:id/react` endpoints lacked authentication. In post creation, `userId` was read from the request body, leading to an Insecure Direct Object Reference (IDOR) where any user could create a post claiming to be any other user.
**Learning:** Controllers must use `req.user.id` (from auth middleware) instead of request body parameters for resource creation to prevent IDOR vulnerabilities. Endpoints that modify state or create resources must be protected by authentication middleware. Tests require `testSetup.js` to be imported before `src/app.js` to ensure the mocked SQLite database and test environment variables (like `JWT_SECRET`) are initialized before models are loaded.
**Prevention:** Always rely on authenticated session state (e.g., `req.user.id`) rather than user input for authorization decisions. Apply `authenticateToken` middleware to all sensitive routes. Ensure test setup logic runs before application initialization in test files.
