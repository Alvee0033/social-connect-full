## 2024-05-24 - [Hardcoded JWT Secret Fallback]
**Vulnerability:** The authentication middleware used a hardcoded fallback secret (`'your-super-secret-jwt-key-change-in-production'`) if `process.env.JWT_SECRET` was missing. This allows attackers to forge valid JWTs if the environment variable isn't correctly configured in production. Additionally, the JWT verification expected `decoded.userId`, but the token was generated using `id`.
**Learning:** Hardcoded fallbacks for cryptographic secrets are extremely dangerous, as developers might unknowingly rely on them when environment variables fail to load. Inconsistent token payloads lead to logic errors and potential authentication bypasses or failures.
**Prevention:** Never provide a default fallback for sensitive secrets. The application should fail fast (e.g., return a 500 error or crash on startup) if crucial configuration is missing. Ensure token generation and verification logic are aligned, ideally using shared constants or types.
