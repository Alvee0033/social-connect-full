## 2024-05-18 - [CRITICAL] Fix IDOR in Post Creation and Auth Middleware Bug
**Vulnerability:** Found a Critical Insecure Direct Object Reference (IDOR) vulnerability in `POST /api/posts` where the user ID of the post creator was taken from the request body (`req.body.userId`). This allowed any user to create a post as if they were someone else. In addition, the authentication middleware (`auth.middleware.js`) was attempting to fetch the user based on `decoded.userId`, but the JWT was actually signed with `id`, meaning the middleware always failed to retrieve the user object and only populated minimal info. Finally, `req.user` was setting `username` instead of `displayName`, causing down-stream logic to break.
**Learning:** Never trust the client to provide the identity of the action creator. Always derive the acting user's identity from a secure source, like a verified session token or JWT payload (`req.user.id`). Furthermore, closely couple token generation logic and verification logic to ensure payload keys (`id` vs `userId`) perfectly align.
**Prevention:** In API endpoints involving resource creation, retrieve the authoritative user ID from `req.user` rather than trusting values in the request body. Add security integration tests to confirm that changing a user ID in the payload does not successfully impersonate them.
